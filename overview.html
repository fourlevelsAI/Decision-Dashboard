What this is

This is a decision accountability system. It forces explicit thinking before commitment and enforces review after outcomes appear. If decisions aren’t reviewed, the system is failing — even if execution looks “busy”.

Risk Signal (system)
A decision is flagged when it’s likely to create downstream damage: Impact = High OR Confidence < 60 OR High impact + Guardrails missing OR Review is overdue OR Status is “Ignored”.
Review Discipline
“Due soon” = review scheduled within 14 days. “Overdue” = review date passed and status is not Reviewed / Rejected. The loop must close with: Outcome + Learning.
Why People Decisions
Hiring, compensation, and org design decisions compound. One wrong hire can distort execution, culture, and cash. This system turns “gut calls” into auditable commitments.
Testers Count
Approximate unique browsers via a lightweight public counter API + local “seen” flag per browser. It’s a directional adoption signal, not analytics-grade truth.
What “Good” Looks Like
Fewer high-risk decisions. Higher guardrail coverage. Reviews completed on time. If risk stays high, you’re shipping decisions without constraints.
Anti-Rationalization Rule
A decision without a review date is a decision you plan to rationalize later. No review date → the loop is intentionally left open.

The point isn’t logging. The point is learning. The system should make leadership judgment visible — to you first.
